**Product Requirements Document v1.1**
**작성일: 2026-06-12 | 최종수정: 2026-06-12**

## 1. 개요

### 1.1 제품 목적

늘사랑교회의 교인 정보, 예배 출석, 목장, 헌금, 목양 데이터를 통합 관리하는 웹 기반 시스템. 기존 엑셀·수기 관리의 비효율을 해소하고, 권한별로 필요한 정보에 접근할 수 있는 환경을 제공한다.

### 1.2 제품 범위

- 공개 랜딩 페이지 (교회 홈페이지)
- 관리자 대시보드 (로그인 필요)

### 1.3 기술 스택

| 구분      | 기술                                          |
| --------- | --------------------------------------------- |
| Frontend  | Next.js 15 (App Router), React 19, TypeScript |
| Styling   | Tailwind CSS                                  |
| Database  | Neon Postgres (Vercel Marketplace)            |
| ORM       | Drizzle ORM                                   |
| Auth      | NextAuth.js v5                                |
| 배포      | Vercel                                        |
| 파일 처리 | Vercel Blob (엑셀 업로드)                     |

## 3. 기능 요구사항

### 3.1 인증

#### 3.1.1 로그인

- 아이디(username) + 비밀번호 로그인
- 로그인 성공 시 역할에 따라 대시보드로 리다이렉트
- 세션 만료 시 자동 로그아웃 (1시간)

#### 3.1.2 계정 관리 (관리자 전용)

- 계정 생성 / 수정 / 비활성화
- 역할 변경 (관리자 / 목회자 / 목자)
- 목자 계정 생성 시 담당 목장 배정 필수 (데이터 접근 범위 결정)

---

### 3.2 통계 홈 (대시보드 메인)

접속 시 가장 먼저 보이는 요약 화면.

| 위젯           | 내용                                   | 권한          |
| -------------- | -------------------------------------- | ------------- |
| 전체 교인 수   | 총 성도 수, 이번 달 신규 등록          | 관리자·목회자 |
| 연령대 분포    | 연령대별 인원 막대/도넛 차트           | 관리자·목회자 |

### 3.3 성도 관리 모듈

#### 3.3.1 성도 목록

- 검색: 이름, 연락처
- 필터: 목장, 성별, 연령대, 등록연도, 활동상태(활동/휴면/이적)
- 정렬: 이름, 등록일, 목장
- 테이블 컬럼: 이름, 성별, 생년월일, 목장, 연락처, 등록일, 상태

#### 3.3.2 성도 상세 (탭 구조)

**기본정보 탭**

- 이름, 성별, 생년월일, 사진
- 연락처 (전화, 이메일)
- 주소
- 등록일, 세례일, 세례 여부
- 소속 목장
- 활동 상태 (활동 / 휴면 / 이적)
- 비고

**가족정보 탭**

- 직접 입력 가족 구성원 목록 (이름, 관계, 연락처, 비고)
- 성도 간 가족 관계 연결 (예: 홍길동 ↔ 홍길순 = 부부) — 연결 시 상대방 프로필에도 반영

**출석 탭**

- 예배별 출석 기록 (날짜, 예배 종류, 출석 여부)
- 성경공부 모임 참여 기록
- 통계: 최근 3개월 참석률, 참석 예배 종류 분포

**헌금 탭** (관리자·목회자만 접근)

- 헌금 내역 목록 (날짜, 종류, 금액)
- 헌금 종류별 참여 현황

**목양 탭**

- 기도제목 목록 (등록일, 제목, 내용, 응답 여부, 응답 코멘트)
- 심방 기록 목록 (날짜, 담당자, 내용)
- VIP 전도대상 명단 (이름, 관계, 연락처, 진행 상태)

#### 3.3.3 CRUD

- 성도 등록 / 수정 / 삭제 (삭제는 소프트 딜리트)
- **엑셀 일괄 업로드** (아래 3.8 참조)

---

### 3.4 예배·출석 모듈

#### 3.4.1 예배 종류 관리

- 예배 종류 CRUD (이름, 요일, 시간)
- 기본 제공: 주일 1부, 주일 2부, 수요예배, 금요기도회, 새벽기도

#### 3.4.2 출석 입력

- 날짜 + 예배 선택 후 성도 목록 체크인 방식
- 개별 성도 출석 기록 직접 CRUD도 가능

#### 3.4.3 출석 현황 뷰

- **예배별 출석률**: 예배 종류 × 날짜별 출석 인원 / 비율
- **성도별 참석률**: 성도 × 기간별 출석률, 참석 예배 종류 현황

#### 3.4.4 CRUD + 엑셀 업로드

### 3.5 목장 모듈

#### 3.5.1 목장 관리

- 목장 CRUD (목장명, 목자 배정, 소속 성도)
- 성도의 목장 이동 이력 기록

#### 3.5.2 성경공부 모임 관리 (목장 귀속)

- 성경공부 모임 CRUD (모임명, 소속 목장, 요일, 담당자)
- 목장별 성경공부 모임 조회
- 성경공부 모임 출석 체크인 (날짜별)

#### 3.5.3 목장모임 출석

- 목장모임 날짜별 출석 체크인
- CRUD

#### 3.5.4 목장 현황 뷰

- **목장별 성도 목록 및 출석률**
- **성도별 목장모임 참석률**
- **성도별 성경공부 참석률** (목장 소속 모임 기준)
- 목자: 본인 목장만 조회 가능

#### 3.5.5 CRUD + 엑셀 업로드

- 목장 명단 엑셀 업로드
- 성경공부 모임 출석 엑셀 업로드

### 3.6 헌금 모듈

#### 3.6.1 헌금 종류 관리

- 헌금 종류 CRUD
- **기본 4종 (확정)**: 십일조, 주일헌금, 감사헌금, 선교헌금
- 관리자가 종류명 추가·수정·삭제 가능

#### 3.6.2 헌금 내역

- 성도별 헌금 기록 CRUD (날짜, 종류, 금액)

#### 3.6.3 헌금 현황 뷰

- **헌금 종류별 참여도**: 각 헌금별 참여 인원 및 비율
- **성도별 헌금 참여도**: 몇 가지 항목에 참여하는지 (0~4종류)
- **월별·연별 추이**: 종류별 참여 인원 라인 차트
- 금액 정보는 관리자·목회자만 열람 가능

#### 3.6.4 CRUD + 엑셀 업로드

---

### 3.7 목양 모듈

#### 3.7.1 기도제목

- CRUD: 성도 연결, 제목, 내용, 등록일
- 응답 여부 토글 (미응답 / 응답됨)
- 응답 코멘트: 응답 여부를 '응답됨'으로 변경 시 코멘트 입력 가능
- 목록에서 미응답 건 우선 표시

#### 3.7.2 심방 기록

- CRUD: 성도 연결, 날짜, 담당자, 심방 내용
- 목록: 날짜 내림차순, 성도·담당자 필터

#### 3.7.3 VIP 전도대상 명단

- CRUD: 담당 성도 연결, VIP 이름, 관계, 연락처, 진행 상태
- 진행 상태: 기도 중 / 접촉 / 전도 중 / 등록
- 등록 상태로 변경 시 성도 등록으로 자동 연결 옵션 제공

#### 3.7.4 CRUD + 엑셀 업로드

### 3.8 엑셀 업로드 공통 규격

모든 모듈의 엑셀 업로드는 동일한 UX 흐름을 따른다.

```
① 템플릿 다운로드   → 각 모듈별 양식(.xlsx) 제공
② 파일 선택         → .xlsx, .xls 지원, 최대 10MB
③ 파싱 미리보기     → 업로드 전 테이블로 데이터 확인
④ 오류 표시         → 누락 필수값, 형식 오류 행 빨간색 하이라이트 + 사유 툴팁
⑤ 일괄 저장         → 오류 행 제외 저장 또는 전체 수정 후 저장 선택 가능
⑥ 결과 리포트       → 성공 N건 / 실패 N건 / 중복 스킵 N건
```

**모듈별 템플릿 컬럼**

| 모듈     | 필수 컬럼                    | 선택 컬럼                         |
| -------- | ---------------------------- | --------------------------------- |
| 성도     | 이름, 성별, 생년월일         | 연락처, 등록일, 목장, 세례여부 등 |
| 출석     | 날짜, 예배종류, 성도명       | 비고                              |
| 목장     | 목장명, 목자명, 성도명       |                                   |
| 헌금     | 날짜, 성도명, 헌금종류, 금액 | 비고                              |
| 기도제목 | 성도명, 제목, 내용           | 등록일                            |
| VIP      | 담당성도명, VIP이름, 관계    | 연락처, 진행상태                  |

## 4. 비기능 요구사항

### 4.1 성능

- 페이지 초기 로딩 3초 이내
- 엑셀 업로드 처리 500행 이내 30초 이내

### 4.2 보안

- 모든 API 라우트 서버사이드 세션 검증
- 역할 기반 접근 제어(RBAC) DB 레벨(Row Level Security) 적용
- 헌금 데이터 접근 로그 기록
- HTTPS 강제 적용 (Vercel 기본 제공)

### 4.3 반응형

- 데스크톱 우선 설계 (대시보드)
- 태블릿 지원 (목자용 모바일 출석 체크인 고려)

## 5. 데이터 모델 요약

```
members (성도)
  ├── family_members (직접 입력 가족)
  ├── member_relations (성도 간 가족 관계)
  ├── attendance_records (예배 출석)
  ├── bible_study_records (성경공부 참석)
  ├── offering_records (헌금 내역)
  ├── prayer_requests (기도제목)
  │     └── answer_comment (응답 코멘트, prayer_requests 내 필드)
  ├── pastoral_visits (심방 기록)
  └── vip_list (VIP 전도대상)

cell_groups (목장)
  ├── members (소속 성도)
  ├── cell_meeting_records (목장모임 출석)
  └── bible_study_groups (성경공부 모임 — 목장 귀속)
        └── bible_study_records (성경공부 참석)

services (예배 종류)
  └── attendance_records

offering_types (헌금 종류: 십일조·주일헌금·감사헌금·선교헌금)
  └── offering_records

users (계정)
  ├── username (로그인 아이디, unique)
  ├── role: admin | pastor | shepherd
  └── cell_group_id (shepherd일 때 담당 목장 — 데이터 접근 범위 결정)
```

## 6. 구현 단계

| 단계      | 내용          | 포함 기능                                                                        |
| --------- | ------------- | -------------------------------------------------------------------------------- |
| **1단계** | 기반 세팅     | Next.js + TypeScript 세팅, 랜딩페이지 마이그레이션, Neon DB 연결, Drizzle 스키마 |
| **2단계** | 인증·권한     | NextAuth.js 로그인, 역할별 라우트 보호, 대시보드 레이아웃                        |

| **4단계** | 예배·목장     | 예배 출석 CRUD, 목장 구성, 목장모임 출석, 통계 뷰                                |
| **5단계** | 헌금·목양     | 헌금 내역·통계, 기도제목(응답), 심방, VIP 명단                                   |
| **6단계** | 통계 대시보드 | 차트, 연령 분포, 전체 현황 요약, 마무리                                          |
